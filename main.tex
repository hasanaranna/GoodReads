\chapter{Introduction}

The Goodreads project aims to provide a comprehensive platform for book lovers to track, rate, and discover new reading material. This documentation focuses specifically on the Book Tracking and Shelving module. This module acts as the core engine responsible for managing user libraries, reading states, and activity history. 

To deliver a seamless user experience, the proposed system architecture leverages the Google Books API to handle robust book search queries and metadata retrieval. Concurrently, Supabase is utilized as the primary backend to ensure secure user authentication and persistent database management for all user-generated data.

\vspace{0.5cm}

The core features implemented and detailed within this system design include:

\begin{itemize}
    \item \textbf{Bookshelf Management:} Allows users to organize their personal libraries into distinct categories, specifically tracking books they are currently reading, have completed, or want to read in the future.
    \item \textbf{Reading Progress:} Enables users to log page counts to track and update their ongoing progress through a specific book.
    \item \textbf{Ratings \& Reviews:} Provides functionality for users to assign star ratings to books and publish, edit, or delete detailed written reviews.
    \item \textbf{Activity Log:} Maintains a comprehensive history of user actions across the platform, allowing users to view their past interactions and reading history.
\end{itemize}

\noindent \textbf{Project Scope Note:} Among the functionalities listed above, the \textbf{Bookshelf Management} feature is our specifically assigned feature that we will demonstrate for this evaluation.

\section{Deployment and DevOps}
\subsection{Hosting and Cloud Platforms}

The entire application, including both the frontend and backend, is hosted on a single DigitalOcean droplet, ensuring a streamlined deployment process.
The database is managed through Supabase, which provides a fully hosted PostgreSQL instance and simplifies connection handling. 
No additional cloud services were required, keeping the deployment straightforward and centralized.
